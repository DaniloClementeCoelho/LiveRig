

# Orientações para o Codex

## Forma de trabalho

- nao execute nada sozinho. me explique como fazer, gerando códigos e me orientando onde colar. se for algo grande, me sugira que voce execute, mas não execute nada sem minha permissao.
- Não faça alterações fora do escopo solicitado.
- Não remova funcionalidades existentes sem autorização.
- Prefira alterações pequenas e incrementais.
- Quando houver dúvida arquitetural, pergunte antes de decidir.

## Padrões de código

- simples e didático ao inves de complexo/robusto/flexivel
- Evite duplicação de código.
- Não introduza novas dependências sem autorização.

## Testes e validação

- Execute os testes relacionados depois de alterar o código.
- Se não for possível executar um teste, explique o motivo.
- Revise o diff antes de considerar a tarefa concluída.

## Documentação

- Atualize os arquivos .md na raiz do projeto quando houver mudanças no pertinentes do projeto.
- Adicione comentários somente quando a intenção do código não for evidente.
- Não crie documentação redundante.

## Segurança

- Nunca inclua senhas, tokens ou credenciais no código.
- Não execute comandos destrutivos sem autorização explícita.

## Comunicação

- Responda em português.
- Seja objetivo.
- Ao concluir, informe:
  1. arquivos modificados;
  2. alterações realizadas;
  3. testes executados;
  4. pendências ou riscos identificados.
